## Create Campaign

Launch an SMS campaign using an agent and contact list.

### Required Parameters

<ParamField body="campaign_id" type="string" required>
  **Required.** Unique campaign identifier.
  
  Format: `{timestamp}x{random_18_digits}`
  
  Example: `1763072374x123456789012345678`
  
  Generate in JavaScript:
  ```javascript
  const campaignId = `${Date.now()}x${Math.floor(Math.random() * 1e18)}`;
  ```
</ParamField>

<ParamField body="campaign_name" type="string" required>
  **Required.** Display name for the campaign.
  
  Max length: 255 characters
  
  Example: `"November 2024 Outreach"`
</ParamField>

<ParamField body="organization_id" type="string" required>
  **Required.** Organization unique_id.
  
  Get from: `POST /v1/organizations` response
  
  Example: `"1763070568417x631086913669115287"`
</ParamField>

<ParamField body="user_id" type="string" required>
  **Required.** User unique_id (campaign creator).
  
  Get from: `POST /v1/organizations/{id}/users` response
  
  Example: `"1763072368418x263803518630956376"`
</ParamField>

<ParamField body="sms_agent_id" type="string" required>
  **Required.** SMS agent unique_id to use for conversations.
  
  Get from: `POST /v1/agents` response
  
  Example: `"1763072369786x980265710064263640"`
  
  The agent's `starting_message` and `general_prompt` will be used for all conversations.
</ParamField>

<ParamField body="teli_house_number" type="string" required>
  **Required.** Phone number to send SMS from.
  
  Format: E.164 (e.g., `+13135559999`)
  
  Must use your organization's `teli_sms_number` (obtained after 10DLC approval).
  
  Get from: `GET /v1/organizations/{id}` â†’ `teli_sms_number`
</ParamField>

<ParamField body="contacts" type="array" required>
  **Required.** Array of contact objects to message.
  
  Minimum: 1 contact
  
  Each contact object:
  - `phone_number` (string, required): E.164 format
  - `first_name` (string, optional): For `{{first_name}}` variable
  - `last_name` (string, optional): For `{{last_name}}` variable
  - `email` (string, optional): Contact's email
  
  Example:
  ```json
  [
    {
      "phone_number": "+15551234567",
      "first_name": "John",
      "last_name": "Doe",
      "email": "john@example.com"
    }
  ]
  ```
</ParamField>

### Optional Parameters

<ParamField body="enable_sms_to_voice" type="boolean" default="false">
  **Optional.** Enable SMS-to-voice transfer feature.
  
  If `true`, users can transfer to a phone call. Requires `voice_agent_id` and `outbound_number`.
</ParamField>

<ParamField body="voice_agent_id" type="string">
  **Optional.** Voice agent ID for call transfers.
  
  Required if `enable_sms_to_voice: true`
</ParamField>

<ParamField body="outbound_number" type="string">
  **Optional.** Phone number for outbound calls.
  
  Required if `enable_sms_to_voice: true`
</ParamField>

<ParamField body="dry_run" type="boolean" default="false">
  **Optional.** If `true`, simulates campaign without sending real SMS.
  
  Useful for testing before launching real campaigns.
</ParamField>

## List Campaigns

Get campaigns for an organization.

<ParamField query="organization_id" path="org_id" type="string" required>
  Organization unique_id
</ParamField>

<ParamField query="user_id" type="string" required>
  User unique_id
</ParamField>

<ParamField query="is_admin" type="boolean" default="false">
  If true, returns all org campaigns. If false, returns only user's campaigns.
</ParamField>

<RequestExample>

```bash cURL
curl -X GET "https://your-api.modal.run/v1/organizations/{org_id}/campaigns?user_id={user_id}&is_admin=true" \
  -H "X-API-Key: your_api_key"
```

</RequestExample>

<ResponseExample>

```json 200 Success
{
  "success": true,
  "count": 5,
  "campaigns": [
    {
      "campaign_id": "1763072374x...",
      "campaign_name": "November Outreach",
      "status": "ACTIVE",
      "total_clients": 25,
      "created_at": "2025-11-13T22:13:00+00:00"
    }
  ],
  "powered_by": "Teli"
}
```

</ResponseExample>

## Delete Campaign

Delete a campaign and all associated data.

<ParamField path="campaign_id" type="string" required>
  Campaign unique_id
</ParamField>

<RequestExample>

```bash cURL
curl -X DELETE https://your-api.modal.run/v1/campaigns/{campaign_id} \
  -H "X-API-Key: your_api_key"
```

</RequestExample>

<ResponseExample>

```json 202 Accepted
{
  "success": true,
  "message": "Campaign deletion started",
  "campaign_id": "1763072374x...",
  "status": "processing",
  "powered_by": "Teli"
}
```

</ResponseExample>

<Warning>
Campaign deletion is permanent and cannot be undone. All conversation history will be archived before deletion.
</Warning>

## Campaign Status Values

| Status | Description |
|--------|-------------|
| `ACTIVE` | Campaign is running |
| `PAUSED` | Campaign is paused |
| `COMPLETED` | Campaign finished |
| `DELETED` | Campaign deleted |

## Best Practices

<AccordionGroup>
  <Accordion title="Phone Number Requirement">
    Use your organization's `teli_sms_number` (obtained after 10DLC approval) as the `teli_house_number`.
  </Accordion>
  
  <Accordion title="Campaign ID Format">
    Generate campaign IDs using: `{timestamp}x{random_18_digits}`
    
    JavaScript example:
    ```javascript
    const campaignId = `${Date.now()}x${Math.floor(Math.random() * 1e18)}`;
    ```
  </Accordion>
  
  <Accordion title="Contact Validation">
    - All phone numbers must be in E.164 format
    - Invalid numbers will be skipped
    - You'll receive a summary of successful/failed contacts
  </Accordion>
</AccordionGroup>
