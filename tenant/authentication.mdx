## API Key Authentication

All API requests require an API key in the `X-API-Key` header.

```bash
X-API-Key: your_api_key_here
```

## Getting Your API Key

Contact your Teli representative or account manager to receive your API key during onboarding.

## Using Your API Key

<CodeGroup>

```bash cURL
curl -X GET https://your-api.modal.run/v1/organizations \
  -H "X-API-Key: your_api_key_here"
```

```javascript JavaScript
const response = await fetch('https://your-api.modal.run/v1/organizations', {
  headers: {
    'X-API-Key': 'your_api_key_here'
  }
});
```

```python Python
import requests

response = requests.get(
    'https://your-api.modal.run/v1/organizations',
    headers={'X-API-Key': 'your_api_key_here'}
)
```

</CodeGroup>

## Security Best Practices

<AccordionGroup>
  <Accordion title="Keep API Keys Secret">
    - Never commit API keys to version control
    - Use environment variables
    - Rotate keys periodically
  </Accordion>
  
  <Accordion title="Server-Side Only">
    - Never expose API keys in client-side code
    - Always make API calls from your backend
  </Accordion>
  
  <Accordion title="Monitor Usage">
    - Track API calls in your logs
    - Set up alerts for unusual activity
  </Accordion>
</AccordionGroup>

## Error Codes

| Status Code | Description |
|-------------|-------------|
| `401` | Missing API key |
| `403` | Invalid API key |
| `400` | Bad request (validation errors) |
| `404` | Resource not found |
| `500` | Internal server error |

## Example Error Response

```json 403 Forbidden
{
  "error": "Invalid API key",
  "powered_by": "Teli"
}
```
