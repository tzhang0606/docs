## Create User

Create a new user in an organization.

<ParamField path="org_id" type="string" required>
  Organization unique_id
</ParamField>

<ParamField body="name" type="string" required>
  User's full name
</ParamField>

<ParamField body="email" type="string" required>
  User's email address (must be unique)
</ParamField>

<ParamField body="permission" type="string" default="user">
  Permission level: `admin` or `user`
</ParamField>

<RequestExample>

```bash cURL
curl -X POST https://your-api.modal.run/v1/organizations/{org_id}/users \
  -H "X-API-Key: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Smith",
    "email": "john@mycompany.com",
    "permission": "admin"
  }'
```

```javascript JavaScript
const response = await fetch(`https://your-api.modal.run/v1/organizations/${orgId}/users`, {
  method: 'POST',
  headers: {
    'X-API-Key': 'your_api_key',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'John Smith',
    email: 'john@mycompany.com',
    permission: 'admin'
  })
});
```

```python Python
response = requests.post(
    f'https://your-api.modal.run/v1/organizations/{org_id}/users',
    headers={'X-API-Key': 'your_api_key'},
    json={
        'name': 'John Smith',
        'email': 'john@mycompany.com',
        'permission': 'admin'
    }
)
```

</RequestExample>

## Permission Levels

| Permission | Can Do |
|-----------|--------|
| `admin` | - See all organization data<br/>- Create/edit/delete campaigns<br/>- Manage all agents and contacts<br/>- Add users |
| `user` | - See only their own campaigns<br/>- Create agents and contacts<br/>- Limited to their own data |

## User Activity Counters

Each user tracks their activity:

- `total_campaigns` - Campaigns they created
- `total_leads` - Contact lists they uploaded
- `total_agents` - Agents they created
- `total_sms_outbound` - SMS they sent
- `total_sms_inbound` - SMS they received

These counters update automatically and never reset.
