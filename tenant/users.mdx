## Create User

Create a new user in an organization.

### Path Parameters

<ParamField path="org_id" type="string" required>
  **Required.** Organization unique_id from URL path.
  
  Example: `1763070568417x631086913669115287`
</ParamField>

### Required Body Parameters

<ParamField body="name" type="string" required>
  **Required.** User's full name.
  
  Max length: 255 characters
  
  Example: `"John Smith"`
</ParamField>

<ParamField body="email" type="string" required>
  **Required.** User's email address.
  
  Must be unique across all users.
  
  Format: Valid email
  
  Example: `"john@mycompany.com"`
</ParamField>

### Optional Parameters

<ParamField body="permission" type="string" default="user">
  **Optional.** User permission level.
  
  Values:
  - `admin`: Full organization access
  - `user`: Limited to own resources
  
  Default: `"user"`
</ParamField>

<RequestExample>

```bash cURL
curl -X POST https://your-api.modal.run/v1/organizations/{org_id}/users \
  -H "X-API-Key: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Smith",
    "email": "john@mycompany.com",
    "permission": "admin"
  }'
```


## Permission Levels

| Permission | Can Do |
|-----------|--------|
| `admin` | - See all organization data<br/>- Create/edit/delete campaigns<br/>- Manage all agents and contacts<br/>- Add users |
| `user` | - See only their own campaigns<br/>- Create agents and contacts<br/>- Limited to their own data |

## User Activity Counters

Each user tracks their activity:

- `total_campaigns` - Campaigns they created
- `total_leads` - Contact lists they uploaded
- `total_agents` - Agents they created
- `total_sms_outbound` - SMS they sent
- `total_sms_inbound` - SMS they received

These counters update automatically and never reset.
