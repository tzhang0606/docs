---
title: 'Create 10DLC Campaign'
api: 'POST /v1/10dlc/campaign'
description: 'Submit campaign for carrier approval'
---

Create a 10DLC campaign registration with sample messages and compliance details.

<Warning>
Campaign must include at least 5 sample messages and clear opt-in/opt-out messaging.
</Warning>

## Authorizations

<ParamField header="X-API-Key" type="string" required>
  Authentication header
</ParamField>

## Body

### Required Fields

<ParamField body="brand_id" type="string" required>
  Brand ID from brand registration (e.g., `B4KRR4F`)
</ParamField>

<ParamField body="subgroup_id" type="string" required>
  SubGroup ID to associate the campaign with
</ParamField>

<ParamField body="campaign_type" type="string" required>
  Campaign type: `STANDARD` or `LOW_VOLUME`
</ParamField>

<ParamField body="use_cases" type="array" required>
  Array of use cases (up to 5). Options:
  - `CUSTOMER_CARE`
  - `2FA`
  - `ACCOUNT_NOTIFICATION`
  - `DELIVERY_NOTIFICATION`
  - `FRAUD_ALERT`
  - `MARKETING`
  - `MIXED`
  - `POLLING_VOTING`
  - `PUBLIC_SERVICE_ANNOUNCEMENT`
  - `SECURITY_ALERT`
</ParamField>

<ParamField body="description" type="string" required>
  Campaign description (40-4096 characters). Describes what messages will be sent.
</ParamField>

<ParamField body="message_flow" type="string" required>
  How subscribers opt-in (40-4096 characters). Describes the consent process.
</ParamField>

<ParamField body="phone_number_ids" type="array" required>
  Array of phone number UUIDs to assign to this campaign. Get IDs from `GET /v1/voice/phone-numbers`.
</ParamField>

<ParamField body="privacy_policy_link" type="string" required>
  URL to your privacy policy
</ParamField>

<ParamField body="terms_and_conditions_link" type="string" required>
  URL to your terms and conditions
</ParamField>

### Sample Messages (All 5 Required)

<ParamField body="sample1" type="string" required>
  Sample message 1 (20-1024 characters). Must include opt-out language.
</ParamField>

<ParamField body="sample2" type="string" required>
  Sample message 2. Must include opt-out language.
</ParamField>

<ParamField body="sample3" type="string" required>
  Sample message 3. Must include opt-out language.
</ParamField>

<ParamField body="sample4" type="string" required>
  Sample message 4. Must include opt-out language.
</ParamField>

<ParamField body="sample5" type="string" required>
  Sample message 5. Must include opt-out language.
</ParamField>

### MMS Media (Optional)

<ParamField body="mms_media_uuid" type="string[]">
  Array of MMS media UUIDs from the [Upload Campaign MMS](/partner-api/10dlc/campaign-mms-upload) endpoint. Include if your campaign sends MMS messages with images or logos.
</ParamField>

### Compliance Options

<ParamField body="direct_lending" type="boolean">
  Is this for direct lending? Default: `false`
</ParamField>

<ParamField body="age_gated" type="boolean">
  Age-restricted content? Default: `false`
</ParamField>

<ParamField body="subscriber_optin" type="boolean">
  Requires opt-in? Default: `true`
</ParamField>

<ParamField body="subscriber_optout" type="boolean">
  Has opt-out? Default: `true`
</ParamField>

<ParamField body="subscriber_help" type="boolean">
  Has help keywords? Default: `true`
</ParamField>

<ParamField body="number_pool" type="boolean">
  Uses number pooling? Default: `false`
</ParamField>

<ParamField body="embedded_link" type="boolean">
  Messages contain links? Default: `false`
</ParamField>

<ParamField body="embedded_phone" type="boolean">
  Messages contain phone numbers? Default: `false`
</ParamField>

### Keywords and Messages

<ParamField body="optin_keywords" type="string">
  Opt-in keywords. Default: `"START, YES, SUBSCRIBE"`
</ParamField>

<ParamField body="optout_keywords" type="string">
  Opt-out keywords. Default: `"STOP, QUIT, UNSUBSCRIBE, END"`
</ParamField>

<ParamField body="help_keywords" type="string">
  Help keywords. Default: `"HELP, INFO"`
</ParamField>

<ParamField body="help_message" type="string">
  Message sent when user requests help (20-320 characters)
</ParamField>

<ParamField body="optin_message" type="string">
  Message sent when user opts in (e.g., "Welcome! You've now opted-in...")
</ParamField>

<ParamField body="optout_message" type="string">
  Message sent when user opts out
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the campaign was submitted successfully
</ResponseField>

<ResponseField name="campaign_id" type="string">
  Campaign registration ID (e.g., `C1OTSR2`)
</ResponseField>

<ResponseField name="status" type="string">
  Campaign status: `PENDING`, `ACTIVE`, `REJECTED`
</ResponseField>

<ResponseField name="carrier_qualification" type="array">
  Carrier approval status for each carrier (AT&T, T-Mobile, Verizon, etc.)
</ResponseField>

## Phone Number Assignment

When you create a campaign:
1. The specified phone numbers are automatically assigned to the campaign
2. Assigned phone numbers are marked as unavailable for other campaigns
3. Each phone number can only be used for one active campaign

## Next Steps

After creating a campaign:
1. Save the `campaign_id` for tracking
2. Wait for carrier approval (1-2 business days)
3. Once approved, you can start sending messages from the assigned phone numbers
4. Check campaign status via `GET /v1/10dlc/campaigns`
