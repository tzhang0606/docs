# Generate Campaign Content

Auto-generate all the required content for a 10DLC campaign registration based on a registered brand and selected use cases. Returns the campaign description, message flow (consent text), sample messages, opt-in/opt-out messages, and HELP message â€” ready to submit.

## Endpoint

```
POST /v1/10dlc/campaign/generate-content
```

## Authentication

<ParamField header="X-API-Key" type="string" required>
  Your API key
</ParamField>

## Request Body

<ParamField body="brand_id" type="string" required>
  The brand ID to generate content for (e.g., `B4KRR4F`)
</ParamField>

<ParamField body="use_cases" type="string[]" required>
  Array of use case identifiers. Available values:
  
  `2FA`, `ACCOUNT_NOTIFICATION`, `CUSTOMER_CARE`, `DELIVERY_NOTIFICATION`, `FRAUD_ALERT`, `HIGHER_EDUCATION`, `MARKETING`, `POLLING_VOTING`, `PUBLIC_SERVICE`, `SECURITY_ALERT`
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the generation was successful
</ResponseField>

<ResponseField name="description" type="string">
  Auto-generated campaign description for the registration form
</ResponseField>

<ResponseField name="message_flow" type="string">
  Complete opt-in consent text with checkboxes. Includes a separate marketing checkbox if `MARKETING` is in the use cases.
</ResponseField>

<ResponseField name="sample_messages" type="string[]">
  Up to 5 sample messages based on the selected use cases (2 per use case)
</ResponseField>

<ResponseField name="sample1" type="string">
  First sample message (ready for campaign create)
</ResponseField>

<ResponseField name="sample2" type="string">
  Second sample message
</ResponseField>

<ResponseField name="sample3" type="string">
  Third sample message
</ResponseField>

<ResponseField name="sample4" type="string">
  Fourth sample message
</ResponseField>

<ResponseField name="sample5" type="string">
  Fifth sample message
</ResponseField>

<ResponseField name="optin_message" type="string">
  The message sent to users when they opt in
</ResponseField>

<ResponseField name="optout_message" type="string">
  The message sent to users when they opt out (reply STOP)
</ResponseField>

<ResponseField name="help_message" type="string">
  The message sent to users when they reply HELP
</ResponseField>

<ResponseField name="optin_keywords" type="string">
  Keywords that trigger opt-in (e.g., `START, YES, SUBSCRIBE`)
</ResponseField>

<ResponseField name="optout_keywords" type="string">
  Keywords that trigger opt-out (e.g., `STOP, UNSUBSCRIBE, CANCEL, END, QUIT`)
</ResponseField>

<ResponseField name="help_keywords" type="string">
  Keywords that trigger the help message (e.g., `HELP, INFO`)
</ResponseField>

<ResponseField name="brand" type="object">
  Brand details used for content generation

  <Expandable title="brand properties">
    <ResponseField name="company_name" type="string">Company display name</ResponseField>
    <ResponseField name="vertical" type="string">Industry vertical code</ResponseField>
    <ResponseField name="industry" type="string">Human-readable industry name</ResponseField>
    <ResponseField name="optin_link" type="string">Opt-in page URL</ResponseField>
    <ResponseField name="website" type="string">Company website</ResponseField>
    <ResponseField name="privacy_policy_link" type="string">Privacy policy URL</ResponseField>
    <ResponseField name="terms_and_conditions_link" type="string">Terms and conditions URL</ResponseField>
  </Expandable>
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X POST "https://api.teli.ai/v1/10dlc/campaign/generate-content" \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "brand_id": "B4KRR4F",
    "use_cases": ["CUSTOMER_CARE", "ACCOUNT_NOTIFICATION", "MARKETING"]
  }'
```

```python Python
import requests

response = requests.post(
    'https://api.teli.ai/v1/10dlc/campaign/generate-content',
    headers={'X-API-Key': 'YOUR_API_KEY'},
    json={
        'brand_id': 'B4KRR4F',
        'use_cases': ['CUSTOMER_CARE', 'ACCOUNT_NOTIFICATION', 'MARKETING']
    }
)
data = response.json()
```
</CodeGroup>

## How to Use the Generated Content

The generated fields map directly to the [Create Campaign](/partner-api/10dlc/campaign-create) endpoint:

| Generated Field | Campaign Create Field |
|---|---|
| `description` | `description` |
| `message_flow` | `messageFlow` |
| `sample_messages[0]` | `sample1` |
| `sample_messages[1]` | `sample2` |
| `sample_messages[2]` | `sample3` |
| `sample_messages[3]` | `sample4` |
| `sample_messages[4]` | `sample5` |
| `optin_message` | `optinMessage` |
| `optout_message` | `optoutMessage` |
| `help_message` | `helpMessage` |
| `optin_keywords` | `optin_keywords` |
| `optout_keywords` | `optout_keywords` |
| `help_keywords` | `help_keywords` |

## Available Use Cases

| Value | Description |
|---|---|
| `2FA` | Two-factor authentication codes |
| `ACCOUNT_NOTIFICATION` | Account status updates and alerts |
| `CUSTOMER_CARE` | Customer support and service messages |
| `DELIVERY_NOTIFICATION` | Shipping and delivery updates |
| `FRAUD_ALERT` | Fraud detection and prevention alerts |
| `HIGHER_EDUCATION` | University and college notifications |
| `MARKETING` | Promotional offers and marketing messages |
| `POLLING_VOTING` | Polling, voting, and survey messages |
| `PUBLIC_SERVICE` | Public service announcements |
| `SECURITY_ALERT` | Security breach and password reset alerts |
