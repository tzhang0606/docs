---
title: 'Create SMS Agent'
api: 'POST /v1/agents'
description: 'Create a reusable SMS conversation agent'
---

Create an SMS agent for text-based AI conversations. SMS agents are simple to set up and support automated drip campaigns.

<Info>
SMS agents have no additional charges beyond standard SMS costs per message.
</Info>

## Authorizations

<ParamField header="X-API-Key" type="string" required>
  Authentication header containing your API key.
</ParamField>

## Required Fields

<ParamField body="agent_type" type="string" required>
  Must be `"sms"` for SMS agents.
</ParamField>

<ParamField body="agent_name" type="string" required>
  Display name for the agent.
  
  Example: `"Sales Follow-up Agent"`
</ParamField>

<ParamField body="starting_message" type="string" required>
  First message sent to contacts. Supports variables: `{{first_name}}`, `{{last_name}}`
  
  Example: `"Hi {{first_name}}, interested in our services?"`
</ParamField>

<ParamField body="prompt" type="string" required>
  AI behavior instructions. Tells the agent how to respond.
  
  Example: `"You are a sales agent for Acme Corp. Ask users about their needs, budget, and timeline. Be friendly and helpful."`
</ParamField>

<ParamField body="organization_id" type="string" required>
  Organization unique_id.
  
  Example: `"1763070568417x631086913669115287"`
</ParamField>

<ParamField body="user_id" type="string" required>
  User unique_id (agent creator).
  
  Example: `"1763072368418x263803518630956376"`
</ParamField>

## Optional Fields

<ParamField body="extraction_fields" type="array">
  Fields to extract from conversations using AI. The AI will analyze conversations and extract these data points.
  
  Example:
  ```json
  ["credit_score", "loan_amount", "annual_income", "interested", "timeline"]
  ```
  
  <Tip>Use descriptive field names like `credit_score` instead of abbreviations like `cs`.</Tip>
</ParamField>

<ParamField body="drip_schedule" type="array">
  Automated follow-up messages if the contact doesn't respond.
  
  Each item: `{ "delay_hours": number, "type": "ai"|"custom", "message": string }`
  
  - `delay_hours`: Hours to wait before sending
  - `type`: `"ai"` for AI-generated follow-up, `"custom"` for your specific message
  - `message`: Required if type is `"custom"`
  
  Example:
  ```json
  [
    {"delay_hours": 24, "type": "ai"},
    {"delay_hours": 72, "type": "custom", "message": "Hi {{first_name}}, just following up!"},
    {"delay_hours": 168, "type": "custom", "message": "Last chance - let me know if I can help!"}
  ]
  ```
</ParamField>

## Response Fields

<ResponseField name="agent_id" type="string">
  Agent identifier. Use this when creating SMS campaigns.
</ResponseField>

<ResponseField name="agent_type" type="string">
  Always `"sms"` for SMS agents.
</ResponseField>

<ResponseField name="has_drip_schedule" type="boolean">
  Whether automated follow-ups are configured.
</ResponseField>

<ResponseField name="has_extraction_fields" type="boolean">
  Whether data extraction is configured.
</ResponseField>

<ResponseField name="extraction_fields" type="array">
  List of fields that will be extracted from conversations.
</ResponseField>

## Extraction Fields Best Practices

- **Use descriptive names**: `credit_score` instead of `cs`
- **Keep fields focused**: Extract specific data points like `loan_amount`, `timeline`
- **Common fields**: `interested`, `budget`, `timeline`, `email`, `best_time_to_call`
- The AI will extract these values as contacts provide them in conversation

## Drip Schedule Best Practices

- **24 hours**: Quick check-in, shows attentiveness
- **72 hours**: Provide value or new information  
- **168 hours (1 week)**: Final gentle follow-up
- Use `type: "ai"` for dynamic AI-generated messages
- Use `type: "custom"` for specific messaging you control
- Use `{{first_name}}` for personalization
